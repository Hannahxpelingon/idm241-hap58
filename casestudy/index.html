Case Study

Overview

Microinteractions are small, yet impactful moments integrated in interfaces that shape users' experience and their overall behavior when interacting with the product. This case study concentrates on redesigning a complex microinteraction within an existing interface. The project showcases how micro interaction design can clarify user intent, reduce friction, and create an engaging experience. Through detailed breakdowns of the triggers, rules, feedback, loops and modes, this study highlights how one small design consists of many parts to elevate the overall user journey.

Context and challenge

Background and Project Objective

Create and code a complex micro interaction that includes 5 triggers

Description

For this case study, I chose to redesign an IKEA card carousel. Figure 1.1 showcases its current state. The current design features a horizontal, draggable bar, and. arrow controls to move the cards left and right.

Goal

My initial goal was to make the card clearly feel clickable. Originally, the only cue was the cursor changing to a pointer on hover, which wasn't strong enough to signal interactivity. I also wanted to replace the draggable crossbar with clearer pagination to help users understand which set of cards they were viewing.

Design Process

Identifying microinteration opportunity

The design process began by analyzing the existing state of the IKEA carousel. This initial audit helped me identify where improvements could be made for a more intuitive experience.

Protoyping

I began by prototyping in Figma to determine the structure of the carousel, which include the following:

  • card size and estimated space (subject to change due to responsiveness in code)
  • position of navigation arrows
  • placement of pagination dots
  • hierarchy of text and images
  • applying IKEA's branding
  • creating interactive components

the high fidelity prototypes gave a visual preview of how the carousel would look and feel, providing a foundational plan before coding.

Iteration of the Design

The project progressed through two main iterations: a beta version that implemented four core triggers, and a final version that expanded the design into a complete, complex microinteraction with all five triggers.

Beta Iteration and Changes

The beta iteration showcased the following changes reflected on the first prototype:

  • Carousel card: On hover on, the card shifts to a blue state while an arrow smoothly slides in from the left, subtly pushing the text to the right and signaling interactivity. On hover off, the blue background disappears, the arrow shifts back to the left and disappears, and the text shifts back to it's original position.
  • Clickable dots were added to give users clearer control over which set of cards is displayed.
  • Left and right chevron arrows: The chevron buttons maintained as the main navigation, allowing users to move sequentially through the cards.

Final Iteration and Changes

To complete the final complex microinteraction of five triggers, a “Read More” button was added.

  • Read More button: When hovered, the button transitions in color from back to gray and on hover off, the color reverts from gray to black. This creates a clear signal that it is an actionable element.

Code/Dev

The development stage involved translate the Figma protoype into a fully functional carousel using HTML, CSS, and Javascript for both iterations. I implemented hover states, slide animations, pagination behavior, and the new “Read More” button through a combination of CSS transitions and JavaScript event handling. A challenge occured when Chrome was causing arrows to disappear due to the layout. This was resolved through targeting styling adjustments. The final build delivered the final iteration that ensures that the microinteractions match the intended prototype design.

The Solution

The final solution is a fully coded-out IKEA card carousel that showcases refined micro interactions. The carousel's UX changes focus on integrating a clear and intuitive navigation where users can move through the cards using arrows and pagination dots, receive immediate visual feedback through hover states, background color changes, and the added “Read More” button. Across both the beta and final iterations, I also learned how to accurately document and describe how the interactions perform with complete breakdowns of triggers, rules, feedback, loops and modes. With each iteration, I became more precise in identifying when and what had changed.

The Results

From start to finish, the project ended in a fully functional complex microinteraction that met my initial goals of providing feedback and clarity for the users. One notable challenge occurred during the beta build, when the navigation arrows would disappear in the Chrome browser due to its layout rendering. By debugging the component structure and refining the CSS for the responsiveness, I was able to resolve the issue. Reflecting on the process. I learned the value of iterative design and accuracy in documentation descriptions, especially in defining triggers, feedback, loops, and modes. In the future, I would like to ensure more time for browser testing and ensure that the code is viable across all platforms. Thanks to this project, I learned the importance and practiced the delivery of a detail-focused approach to micro interaction design.
